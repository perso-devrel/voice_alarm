# P38 완료 — 계정 삭제 토스트 + Auth 에러 코드

## 집은 항목
P38 미완료 2개:
- 웹: SettingsPage 계정 삭제 후 에러 토스트 개선
- 백엔드: auth 미들웨어 토큰 만료/무효 시 명확한 에러 메시지

## 변경 파일

### packages/web/src/pages/SettingsPage.tsx
- `alert()` → 인라인 toast 알림으로 교체
- 401/429/일반 에러 각각 다른 메시지 표시
- 계정 삭제 성공 시 `/login`으로 리다이렉트 (reload 대신)
- useRef 초기값 추가 (strict TS 호환)

### packages/backend/src/middleware/auth.ts
- 에러 응답에 `code` 필드 추가 (AUTH_MISSING, AUTH_INVALID_SCHEME, AUTH_EMPTY_TOKEN, AUTH_TOKEN_EXPIRED, AUTH_AUDIENCE_MISMATCH, AUTH_INVALID_ISSUER, AUTH_MALFORMED_TOKEN, AUTH_VERIFICATION_FAILED)
- 빈 토큰 체크 추가
- Authorization 헤더 누락 vs Bearer 스킴 오류 분리

## 검증
- `packages/backend`: tsc --noEmit ✅, vitest 212 tests all pass ✅
- `packages/web`: tsc --noEmit ✅, npm run build ✅

## 주의사항
- 기존 백엔드 테스트에 auth 미들웨어 직접 테스트 없음 (mock으로 우회). 새 에러 코드는 기존 테스트에 영향 없음.
- 클라이언트(모바일/웹)의 401 인터셉터는 error 필드를 사용하지 않으므로 code 추가는 하위 호환.
