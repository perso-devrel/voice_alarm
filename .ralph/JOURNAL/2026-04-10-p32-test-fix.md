## 집은 항목

## 근본 원인
3가지 변경이 누적되면서 기존 테스트가 실제 라우트 동작과 불일치:

1. **UUID 검증 추가 (P10/P12)**: URL 파라미터와 body의 ID 필드에 UUID 형식 검증이 추가됨. 기존 테스트는 'g1', 'f1', 'a1', 'm1' 같은 짧은 ID를 사용해 400 응답을 받음.
2. **페이지네이션 추가 (P9/P13)**: GET 목록 엔드포인트가 `Promise.all`로 count + data 두 쿼리를 실행. 기존 테스트는 mock 1개만 제공해서 count 쿼리에서 `rows[0].total`이 undefined → NaN 또는 빈 배열 반환.
3. **PATCH 응답 개선 (P27)**: accept/reject/update 라우트가 UPDATE 후 변경된 객체를 SELECT로 다시 조회해 반환. 기존 테스트는 이 추가 SELECT에 대한 mock이 없어 500 오류.
